# Contributing to DataMind AI

Thank you for your interest in contributing to our project! To keep the codebase clean and functional, please follow these guidelines:

## 🌿 Branching Strategy
* Always create a feature branch from `main`: `git checkout -b feat/your-feature-name`
* For bug fixes, use: `git checkout -b fix/bug-name`

## 💬 Commit Message Formats
We follow clear semantic commit prefixes:
* `feat:` for new features (e.g., `feat: add interactive charts`)
* `fix:` for bug fixes (e.g., `fix: resolve dashboard rendering issue`)
* `docs:` for documentation changes (e.g., `docs: update readme`)

## 🚀 Pull Request Process
1. Ensure your local build passes via `npm run build` inside the frontend directory before pushing.
2. Submit a Pull Request targeting the `main` branch with a clear summary of your changes.
